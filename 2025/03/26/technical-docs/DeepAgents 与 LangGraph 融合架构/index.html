7. Open WebUI 接入方式

Open WebUI 的接入方式通常有以下几种：

方式 1：把 LangGraph 服务做成 OpenAI 兼容模型提供方

这是最直接的方法。

操作思路

  • 部署 LangGraph API
  • 在 Open WebUI 的模型配置中填入：
    • Base URL: http://langgraph-service:8000/v1
    • API Key: 自定义
  • Open WebUI 使用该 endpoint 进行对话

适用场景

  • 你主要做聊天型 Agent
  • 想快速上线

方式 2：通过 Open WebUI 的 Pipelines / Functions / Custom Backend

如果你的 Open WebUI 版本支持 pipelines 或插件机制，可以：

  • 在 WebUI 端定义一个“模型”
  • 这个模型实际请求你的 LangGraph backend

适用场景

  • 需要多模型管理
  • 需要把多个 Agent 暴露到 UI 中

方式 3：通过反向代理路由

如果你想把多个能力聚合：

  • /v1/chat/completions -> LangGraph
  • /v1/models -> 模型注册中心
  • /api/* -> Agent 管理服务

可通过 Nginx / Traefik / API Gateway 做路径分发。

2）状态持久化

LangGraph 支持 checkpointer 机制。

推荐：

  • 开发：InMemory
  • 生产：PostgresCheckpointer

这样可以：

  • 恢复中断任务

LangGraph 执行时需要将 token 或节点事件实时推送出去。

可以有两种方式：

  • 直接 SSE
  • WebSocket

对 Open WebUI 来说，SSE 更容易兼容。
