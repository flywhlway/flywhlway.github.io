1. 认知重置

1.1 三个时代的 “Codex”，不要混淆

理解 Codex 前，必须先做一次认知清零。历史上 “Codex” 这个名字出现过三次，含义完全不同：

2021 Codex API（已废弃）
OpenAI 基于 GPT-3 微调的代码补全模型，是 GitHub Copilot 早期自动补全功能的动力引擎。这个产品已于 2023 年彻底下线，与本文完全无关。

2025 Codex Web（云端 Agent）
2025 年 5 月随 ChatGPT 推出的云端编程 Agent。你在 chatgpt.com/codex 提交任务，它在隔离的云沙箱里异步完成，然后开一个 Pull Request。适合委托独立 Bug 修复、功能实现等批量任务。

2025 Codex CLI（本文主角）
2025 年 4 月 16 日发布的开源终端编程 Agent，用 Rust 构建，Apache-2.0 协议。截至 2026 年 4 月拥有约 75K GitHub Star、1453 万月 npm 下载、400 万周活跃用户（Sam Altman 确认数据）。这才是与 Claude Code 直接对标的产品，也是本文的全部内容。

一句话定位：Codex CLI = 你终端里的 OpenAI 版 Claude Code，Rust 构建，跑 GPT-5.x 系列模型，内置 OS 级沙箱。

1.2 为什么 Claude Code 用户值得认真学习 Codex

如果你已经用 Claude Code 做出了生产力，这里有几个真实的理由让你投入时间学 Codex：

  • Token 效率：社区测试显示 Codex 完成同等任务消耗的 token 约为 Claude Code 的 25%–33%，对高频使用者意义重大
  • Terminal-Bench 领先：在终端原生任务上 Codex 77.3% vs Claude Code 65.4%（Terminal-Bench 2.0）
  • 沙箱架构：OS 级内核沙箱（macOS Seatbelt / Linux Landlock），审查非信任代码时边界更硬
  • ChatGPT Plus 附带：$20/月的 ChatGPT Plus 订阅已经包含 Codex CLI 额度，无需额外支出
  • AGENTS.md 开放标准：兼容 Cursor、Aider 等工具，不绑定单一平台
  • 互补而非替代：许多团队用 Claude Code 做复杂架构推理，用 Codex 做批量 PR 生成和 CI 集成——两者共存

6. Skills 体系：从入门到自建

6.1 Skills 是什么

Skills 是可复用的行为包：一个 SKILL.md 文件（加可选脚本和资源），告诉 Codex 如何执行某类特定任务。AGENTS.md 定义项目规范，Skills 定义可移植的工作流程。

关键设计原则：渐进式上下文加载

Codex 启动时只加载每个 Skill 的 name 和 description（约占上下文的 2%）。只有当任务匹配时，才加载完整 SKILL.md 正文。这让你可以安装大量 Skills 而不影响性能。

6.2 Skills 目录结构

建议：个人开发者用 ~/.codex/skills/；需要跨工具共享（同时用 Claude Code）时用 ~/.agents/skills/。

8. 沙箱与审批机制：安全地放权给 AI

Codex 的沙箱机制是它最核心的安全设计，也是与 Claude Code 架构最大的差异。

8.1 沙箱架构

macOS: Apple Seatbelt（OS 内核级别）
Linux: Landlock LSM + Bubblewrap (bwrap)（Linux 安全模块）
Windows: Windows 沙箱容器

这意味着即使提示词注入攻击发生，攻击代码也无法突破沙箱边界——这是应用层 Hook 无法提供的保障。
