一、全面介绍 gRPC 协议

1. 基本概念

gRPC（Google Remote Procedure Call）是 Google 开发的高性能、开源的远程过程调用（RPC）框架。它基于 HTTP/2 协议，并默认使用 Protocol Buffers（Protobuf） 作为接口定义语言和序列化格式。

特点：

  • 跨语言支持：官方支持 C++, Java, Python, Go, C#, Node.js 等。
  • 高性能：基于 HTTP/2 支持多路复用、流控、头压缩。
  • 接口清晰：通过 Protobuf 定义服务和消息格式。
  • 自动生成客户端和服务端代码：避免手写大量序列化/反序列化逻辑。
  • 支持双向流：可以实现服务端流、客户端流、双向流。
  • 可集成服务治理：结合 Nacos、Consul、etcd 等注册中心，实现服务发现和负载均衡。

2. 核心概念

  1. Service（服务）

    定义一个服务接口 Greeter，包含 RPC 方法 SayHello。

  2. Message（消息）

    用于定义输入和输出参数。

  3. 通信模式

    • Unary RPC：客户端发送一个请求，服务端返回一个响应。
    • Server Streaming RPC：客户端发送一个请求，服务端返回一个流。
    • Client Streaming RPC：客户端发送一个流，服务端返回一个响应。
    • Bidirectional Streaming RPC：双方发送/接收流，可同时进行。

  4. 传输协议

    • 基于 HTTP/2，支持多路复用。
    • 默认使用 Protobuf 序列化，支持 JSON（grpc-gateway 转换）。

3. gRPC 与微服务集成优势

  • 高效通信：相比 REST，Protobuf 占用带宽少，解析快。
  • 强类型接口：编译期检查，降低运行时错误。
  • 服务治理友好：结合注册中心（Nacos、Consul）实现服务发现与负载均衡。
  • 跨语言互操作：Python 服务与 Java 服务可以无缝通信。
