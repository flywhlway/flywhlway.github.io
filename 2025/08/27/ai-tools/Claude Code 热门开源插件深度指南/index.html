1. 插件体系概览

在深入研究各个项目之前，有必要先厘清 Claude Code 的扩展体系，因为这五个项目在技术层面属于不同的扩展类型，混淆它们是导致”盲目全装”的根源。

Claude Code 的四种扩展原语

2. 五大热门项目深度解析

2.1 everything-claude-code

GitHub: affaan-m/everything-claude-code · ⭐ ~93k（2026年3月）

本质定位：这是一套跨运行时的 Agent Harness 性能优化系统，涵盖 Skills、本能规则（Instincts）、记忆机制、安全加固和研究优先开发理念。

核心组件

  • Skills 库：覆盖 10 种编程语言（Python、TypeScript、Java、Kotlin、Rust 等），含 PyTorch 深度学习、Next.js、BunRuntime 等专项技能
  • Instincts（本能）：持久化行为规则，Claude 在每次任务前自动检查的约束条件
  • AgentShield 安全：1282 条安全规则、102 个规则集，/security-scan 可直接调用
  • 跨平台支持：Claude Code、Cursor、Codex、OpenCode 均适用
  • 选择性安装：通过 install-plan.js 和 install-apply.js 进行清单驱动安装，可只选需要的组件
  • 内部测试：997+ 测试用例持续通过

安装方式

适合谁

  • 需要跨多个 AI 工具（不只 Claude Code）保持一致体验的开发者
  • 有 AI 安全顾虑、需要 AgentShield 防护的团队
  • 做深度学习、多语言工程的开发者

注意事项

  • 组件数量庞大（Skills、Instincts、安全规则等），建议选择性安装，而非全量安装
  • 更新速度快，需定期同步

2.2 claude-code-best-practice

GitHub: shanraisshan/claude-code-best-practice · ⭐ ~93k（2026年3月）

本质定位：这不是一个可以直接安装的插件包，而是 Claude Code 最佳实践的参考实现库和知识图谱。它本身就是用 Claude Code 的最佳方式构建的，是”示范代码库”。

核心内容

  • Command → Agent → Skill 编排架构：以 /weather-orchestrator 为样例，演示多层编排
  • 高密度 Tips 集合：覆盖 Prompting、CLAUDE.md 编写、Hooks、Agents、Commands、Memory、Settings 的最佳实践
  • Hooks 声音通知系统：跨平台钩子 + 配置文件 + ElevenLabs 生成的音效
  • 工作流模式总结：所有主要工作流收敛于 Research → Plan → Execute → Review → Ship

使用方式（关键！）

这个仓库应该作为参考，而不是直接复制粘贴。 官方推荐的使用方式：

对全栈开发者最有价值的技巧摘要

  • 每天重复超过一次的操作 → 写成 Skill 或 Command
  • 用 context: fork 将 Skill 跑在隔离子代理中，主上下文只看最终结果
  • 每个 Skill 建一个 Gotchas 章节，记录 Claude 的失败点
  • description 字段是触发器，不是摘要——要告诉模型”什么时候该启用我”
  • 用 PostToolUse Hook 自动格式化代码，避免 CI 失败

2.3 get-shit-done (GSD)

GitHub: gsd-build/get-shit-done · ⭐ ~38k（2026年3月）

本质定位：这是一个轻量级、元提示（meta-prompting）驱动的、规格优先（spec-driven）开发系统。GSD 的核心哲学是：复杂性在系统里，不在你的工作流里。

工作原理

GSD 的精髓是上下文工程：将每个任务分配给拥有全新 200k 上下文的专用子代理执行，主会话保持在 30-40% 的上下文占用率。

每个任务都会对应一个独立的 git commit，保持干净的历史。

2.4 ComposioHQ / awesome-claude-skills

GitHub: ComposioHQ/awesome-claude-skills · ⭐ ~44.5k（2026年3月）

本质定位：这是一个 Skills 合集目录，分为两个层次：精心整理的各类 Skills 清单，以及 Composio 自家的 connect-apps 插件（可让 Claude 连接 500+ 外部服务）。

两个不同的东西，请勿混淆

① awesome-claude-skills（Skills 目录）

这是一个索引，列出了大量社区贡献的 Skills，按领域分类：

  • 文档处理：docx、pdf、pptx、xlsx
  • 开发：aws-skills、Playwright 自动化、LangSmith 调试
  • 内容创作：content-research-writer、changelog-generator
  • 工具：file-organizer、iOS Simulator、D3 可视化
  • 安全：OWASP 合规、Trail of Bits 安全审计

安装方式：找到你想要的 Skill，直接将对应的 SKILL.md 文件复制到你的 .claude/skills/ 目录。

② connect-apps 插件（Composio 核心功能）

这是一个可以安装的 Claude Code 插件，通过 Composio 管理 Auth，让 Claude 直接操作：Gmail、Slack、GitHub、Notion、Asana、Linear 等 500+ 应用。

安装后粘贴 Composio API Key（在 platform.composio.dev 免费获取），即可让 Claude 给你发邮件、创建 Issue、发 Slack 消息。

对全栈 + AI 开发者最有价值的 Skills

2.5 superpowers

GitHub: obra/superpowers · ⭐ ~40.9k（2026年3月）

本质定位：这是一套软件工程最佳实践方法论的 Skills 集合，已正式入选 Anthropic 官方 Marketplace。它不是功能扩展，而是将”资深工程师的开发方式”注入 Claude 的行为中。

核心七阶段工作流

所有阶段的技能自动触发——Claude 在每次任务前检查相关 Skills 并强制应用，无需手动调用。

核心 Skills

  • brainstorming：结构化反思设计，写代码前先问清楚需求
  • tdd：严格的测试驱动开发，测试永远先于代码
  • code-review：系统性代码审查，Bug 猎手 + 安全审计 + 代码质量多维度
  • debugging：系统性调试方法论，而不是随机尝试
  • planning：将工作分解为 2-5 分钟的可验证微任务
  • 自定义 Skills 创建工具：可以基于这套体系构建团队专属 Skills

安装方式

适合谁

  • 希望系统性提升 AI 生成代码质量的开发者
  • 需要强制 TDD 流程的团队
  • 已有 Claude Code 使用经验，想要”方法论升级”的进阶用户

4. 最佳组合方案推荐

根据不同的开发者画像，推荐三套组合方案。

方案 A：独立开发者 · 快速交付型（推荐）

核心诉求：一个人写代码，要快，要可靠，不想折腾配置。

安装顺序与原因：

  1. GSD（主框架）：管理整个开发生命周期，分阶段、有门控、保持上下文干净
  2. Superpowers（质量内核）：Skills 自动触发 TDD 和代码审查，强制质量基线
  3. ComposioHQ 精选 Skills：按需取用 frontend-design、webapp-testing、docx/pdf/xlsx

搭配逻辑：GSD 负责”做什么、怎么分解”，Superpowers 负责”怎么做好”，ComposioHQ 补充具体的技术域能力。

不安装：everything-claude-code（全量太重）、shanraisshan（仅作参考，不安装）

token 消耗：中等。GSD 子代理 + Superpowers 自动检查 = 比裸用 Claude Code 贵约 2-3x，但产出质量和自主性大幅提升。

方案 B：全栈 AI 应用开发者（适合你的情况）

核心诉求：构建 RAG 应用、微调、本地 LLM 部署，需要深度 AI 工程能力 + 项目管理。

组件选择：

GSD（项目框架）

everything-claude-code（选装 AI/ML 专项）

ComposioHQ connect-apps（外部服务集成）

单独挑选的 Skills（直接复制 SKILL.md）：

  • aws-skills（如用 AWS 部署）
  • webapp-testing（Playwright 自动化）
  • mcp-builder（你肯定会写 MCP Server）
  • prompt-engineering（构建 RAG 时很有用）

参考学习：shanraisshan/claude-code-best-practice（克隆后学习，不安装到项目）

方案 C：团队协作型（追求一致性）

核心诉求：多人团队，统一工具链，代码质量可审计。

团队推荐做法：

  1. 用 Superpowers 作为共享的质量基线，通过 Plugin Marketplace 安装确保一致性
  2. 用 GSD 管理项目规划和执行，所有人使用同一套工作流
  3. 将 shanraisshan/claude-code-best-practice 作为”教材”，让团队成员学习后各自编写项目专属 CLAUDE.md
  4. 根据项目需求从 ComposioHQ awesome-claude-skills 挑选具体 Skills，签入 .claude/skills/ 进行版本管理
