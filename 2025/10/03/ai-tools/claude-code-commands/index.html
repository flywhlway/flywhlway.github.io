CLI 命令

终端中运行的顶级 claude 子命令

15 个命令

claude

启动交互式会话。直接进入 Claude Code REPL 环境，开始与 AI 进行对话式编程协作。

示例

claude

claude “prompt”

以特定提示词启动会话。直接传入问题或指令，Claude 将立即处理而无需等待输入。

示例

claude “解释这个文件中的认证逻辑”

claude -c

常用

恢复最近一次会话。无需重新解释上下文，继续上次未完成的工作。等同于 –continue。

示例

claude -c

claude -r “name”

按名称恢复特定会话。若不带参数则打开会话选择器，列出所有已命名的历史会话。

示例

claude -r “auth-refactor”

claude -p “prompt”

非交互

非交互模式（Print mode）。输出单次响应后退出，适用于脚本自动化和 CI/CD 流水线。

示例

claude -p “检查代码质量” –output-format json

claude –teleport

Max

从云端拉取远程会话到本地。需要 Claude Max 订阅，从 claude.ai/code 获取会话 ID。

示例

claude –teleport pr-review-session-xyz

claude rc

新功能

Remote Control 模式。通过移动端或网页界面远程操控本地 Claude Code 会话。

示例

claude rc

claude config

打开配置管理子命令组。管理全局和项目级别的 Claude Code 设置与偏好项。

示例

claude config list

claude mcp

MCP 服务器管理子命令组。添加、列出、删除和配置 Model Context Protocol 服务器。

示例

claude mcp list

claude update

将 Claude Code 更新到最新版本。等同于 npm update -g @anthropic-ai/claude-code。

示例

claude update

claude –version

显示当前安装的 Claude Code 版本号和构建信息。

示例

claude –version

claude –help

显示帮助信息，列出所有可用的 CLI 标志和子命令的简要说明。

示例

claude –help

claude plugin

新功能

插件市场管理。安装、更新、列出和删除来自 Claude Code 插件生态的扩展程序。

示例

claude plugin install

claude bundle

v2.1.63+

管理 Bundle（项目配置包）。打包和分享 CLAUDE.md、skills、hooks 等项目配置。

示例

claude bundle export my-project

claude doctor

运行安装健康检查。诊断常见配置问题、权限错误和依赖缺失等故障。

示例

claude doctor

⚑

启动标志

运行 claude 命令时可附加的选项参数

20 个标志

–continue / -c

常用

继续最近的对话会话，保持完整的上下文历史，无需重复解释项目背景。

示例

claude –continue

–resume / -r

按名称或 ID 恢复特定会话。省略参数则打开交互式会话选择器。

示例

claude -r “feature-auth”

–print / -p

非交互式打印模式。Claude 响应后立即退出，适合脚本集成和管道处理。

示例

claude -p “审查代码” < main.py

–output-format

设置输出格式。可选值：text（默认）、json、stream-json。非交互模式下常配合使用。

示例

claude -p “…” –output-format json

–model / -m

指定使用的模型。可选 claude-opus-4-6、claude-sonnet-4-6、claude-haiku-4-5 等。

示例

claude -m claude-opus-4-6

–fast

2026

以 Fast Mode 启动。以 2.5x 速度运行 Opus 4.6，适合实时调试，但成本较高。

示例

claude –fast

–add-dir

添加额外的工作目录到会话范围。支持 monorepo 等多项目场景，可多次使用。

示例

claude –add-dir ../shared-lib

–system-prompt

替换默认系统提示词。与 –system-prompt-file 互斥，会覆盖内置能力，谨慎使用。

示例

claude –system-prompt “You are a Python expert…”

–system-prompt-file

从文件加载系统提示词替换默认值。适用于复杂的团队标准化提示词管理。

示例

claude –system-prompt-file ./prompts/team.md

–append-system-prompt

推荐

追加内容到系统提示词末尾（保留内置能力）。大多数场景优先使用追加标志。

示例

claude –append-system-prompt “Always use TypeScript strict mode”

–allowedTools

指定允许使用的工具白名单。格式为逗号分隔的工具名：Read,Write,Bash,Grep 等。

示例

claude -p “…” –allowedTools “Read,Grep,Glob”

–disallowedTools

指定禁止使用的工具黑名单。禁止特定高风险操作，适用于安全敏感环境。

示例

claude –disallowedTools “Bash,Write”

–dangerously-skip-permissions

危险

YOLO 模式：跳过所有权限确认（v2.0.68+）。仅用于隔离的自动化环境，生产环境禁用。

示例

claude –dangerously-skip-permissions -p “…”

–agents

新功能

定义 Agent Team。以 JSON 格式传入子 Agent 配置，用于多 Agent 协同任务。

示例

claude –agents ‘[{“name”:”reviewer”,”model”:”opus”}]’

–max-turns

限制非交互模式下的最大对话轮数，防止自动化任务无限循环消耗 token。

示例

claude -p “…” –max-turns 10

–input-format

设置输入格式。可选 text（默认）或 stream-json，用于程序化输入场景。

示例

claude –input-format stream-json

–verbose

显示详细的调试输出，包括工具调用过程、思考步骤和内部状态，便于问题诊断。

示例

claude –verbose

–debug

启用 debug 模式，输出更底层的诊断信息，包括 API 请求和响应细节。

示例

claude –debug

–no-stream

禁用流式输出，等待完整响应后一次性显示。适用于某些不支持流式显示的终端。

示例

claude –no-stream -p “…”

–inference-geo

企业

指定推理地域（如 us）以满足数据驻留要求。启用后有 1.1× 价格倍率（Opus 4.6+）。

示例

claude –inference-geo us

/

Slash 命令

会话内输入 / 开头使用的交互式命令

24 个命令

/help

常用

显示所有可用 slash 命令，包括自定义 skills 命令和 MCP 服务器命令。输入 / 后持续键入可过滤。

/clear

常用

清空对话历史，重置上下文窗口。切换任务时使用，从干净状态开始新工作。

/compact [instructions]

常用

压缩对话历史以节省上下文。可传入保留重点的指令。上下文超 80% 时使用，压缩即时完成（v2.0.64+）。

示例

/compact 保留认证模块和当前测试失败信息

/context

显示当前上下文窗口使用量。监控 token 消耗，决定何时需要 /compact 或 /clear。

/resume [name]

恢复历史会话。不带参数打开选择器，传入名称直接跳转。等同于 /continue。

示例

/resume auth-refactor

/rename [name]

为当前会话命名。不带参数则根据对话内容自动生成名称，便于后续查找。

示例

/rename feat-oauth-integration

/fork

在当前状态创建会话分支。相当于 git 分支，可以探索不同方案后回退到分叉点。

/rewind

时光回溯：回退到对话或代码的任意检查点。支持仅回退代码或仅回退对话的选项。

/init

分析当前项目并生成 CLAUDE.md 文件。自动识别技术栈、目录结构和编码规范。

/model [name]

常用

切换使用的模型。可在 Haiku（快/省）/ Sonnet（均衡）/ Opus（强力）之间灵活切换。

示例

/model sonnet

/fast

2026

切换 Fast Mode。以 2.5x 速度运行 Opus 4.6，适合实时迭代。注意：开启后之前上下文会按新费率重新计费。

/plan

进入计划模式。Claude 先制定详细执行计划，等待确认后再实施，避免直接修改代码。

/think / ultrathink

启用扩展思考模式（延长推理）。用于需要深度分析的复杂架构问题，会消耗更多 token。

/mcp

查看已连接的 MCP 服务器状态，以及各服务器提供的工具列表和连接情况。

/add-dir

在会话中添加额外工作目录。用于 monorepo 或跨项目操作，无需重启会话。

示例

/add-dir ../packages/shared

/cost

显示当前会话的 API token 消耗和费用估算（API 用户适用）。

/stats

显示用量统计信息（Pro/Max 订阅用户适用）。查看本周消耗情况和配额剩余。

/usage

检查是否接近速率限制，以及通过 /extra-usage 配置超出限制后的处理方式。

/review

请求对当前更改进行代码审查。Claude 将从架构、安全、性能等维度提供专业反馈。

/todos

列出 Claude 在会话中追踪的所有待办事项。确保重要行动项不被遗漏。

/export [filename]

将当前会话导出为文件或剪贴板。便于分享解决方案或生成技术文档。

示例

/export session-notes.md

/doctor

运行安装健康检查，诊断配置问题。遇到异常行为时首先运行此命令。

/keybindings

打开 ~/.claude/keybindings.json 进行键绑定自定义。修改后无需重启即时生效。

/terminal-setup

配置终端以支持所有快捷键（如 Alt+T 扩展思考切换）。首次使用建议运行。

⟁

MCP 服务器管理

claude mcp 子命令，管理 Model Context Protocol 服务器

6 个命令

claude mcp list

列出所有已配置的 MCP 服务器，显示名称、类型、URL/路径和连接状态。

claude mcp add

常用

添加新的 MCP 服务器。支持 URL 类型（SSE）和 stdio 类型，可指定作用域（local/project/global）。

示例

claude mcp add –name github –url https://api.github.mcp.example.com/sse

claude mcp remove

删除指定名称的 MCP 服务器配置。从 settings.json 中移除对应条目。

示例

claude mcp remove github

claude mcp get

查看特定 MCP 服务器的详细配置信息，包括端点、认证方式和可用工具列表。

claude mcp reset-project-choices

重置项目级别的 MCP 服务器使用授权记录。在权限出现问题时用于重新授权。

claude mcp serve

Beta

将 Claude Code 本身作为 MCP 服务器运行，供其他 AI 工具或系统调用其能力。

⌘

键盘快捷键

交互模式下的键盘操作，运行 /terminal-setup 后完整启用

14 个快捷键

Esc

常用

中断/取消当前 Claude 的操作或生成过程，保留已完成的部分输出。

Esc + Esc

打开 Rewind（回退）菜单，可选择仅回退代码更改或仅回退对话历史。

Shift + Tab

常用

循环切换模式：正常模式 → 自动接受（Auto-accept）→ 计划模式。当前模式显示在提示符下方。

Alt + P

切换活动模型（保留输入框内容）。在高性能 Opus 和高效 Sonnet/Haiku 之间快速切换。

Alt + T

切换下一轮的扩展思考（延伸推理）模式。处理复杂架构问题前开启。需先运行 /terminal-setup。

↑ / ↓

在命令历史中导航。快速重用之前发送过的提示词，减少重复输入。

Tab

自动补全文件路径、命令名称和 @mention 引用。与 @ 符号配合尤其高效。

Ctrl + C

中止当前操作或退出当前提示输入。连按两次可退出 Claude Code 会话。

Ctrl + D

发送 EOF 信号，优雅地退出 Claude Code 会话，保存会话状态供后续恢复。

Ctrl + R

在 bash 历史搜索模式下反向搜索命令历史（终端原生功能）。

Ctrl + L

清屏（不清除对话历史）。与 /clear 的区别：仅刷新显示，保留上下文。

@ + filename

常用

直接引用文件或目录。比描述文件路径更快，Claude 精准定位目标文件内容。

示例

@./src/auth/middleware.ts 检查这里的权限验证逻辑

! + command

在 Claude 会话中直接执行 shell 命令（绕过对话模式）。比通过 Claude 运行更省 token。

示例

!npm test
!git status

/ + 过滤文字

在 slash 命令列表中实时过滤。输入 / 后持续键入关键词，快速定位目标命令。

示例

/mod → 筛选出 /model

⚙

配置参考

关键配置文件、环境变量和目录结构速查

8 个条目

CLAUDE.md（项目级）

核心

项目记忆文件，置于项目根目录。存储编码规范、架构说明、技术栈信息，Claude 启动时自动加载。

位置

./.claude/CLAUDE.md 或 ./CLAUDE.md

CLAUDE.md（全局）

全局记忆文件，应用于所有项目。存储个人偏好、通用规范和跨项目一致性设置。

位置

~/.claude/CLAUDE.md

settings.json

主配置文件。管理工具权限白/黑名单、默认模型、MCP 服务器列表和会话偏好。支持项目和全局两个层级。

位置

.claude/settings.json · ~/.claude/settings.json

.claude/skills/

推荐

自定义 Skill 目录（新版推荐方式）。每个 SKILL.md 生成对应 slash 命令，支持子目录和多文件 Bundle。

结构

.claude/skills/review/SKILL.md → /review

.claude/commands/

旧版自定义命令目录（仍可用）。Markdown 文件生成 slash 命令，现已并入 skills 体系但向后兼容。

结构

.claude/commands/review.md → /review

.claude/agents/

新功能

自定义子 Agent 定义目录。Markdown 文件定义专用 Agent（代码审查、调试等），通过 SKILL.md 的 agent 字段引用。

结构

.claude/agents/reviewer.md

hooks 配置

通过 settings.json 配置 hooks 监听器。核心事件：PreToolUse、PostToolUse、Stop、Notification、SubagentStop 等。

位置

settings.json → { “hooks”: { … } }

环境变量

ANTHROPIC_API_KEY（认证）、CLAUDE_MODEL（默认模型）、ANTHROPIC_BASE_URL（代理端点）、CLAUDE_MAX_TOKENS 等。

示例

export ANTHROPIC_API_KEY=”sk-ant-…”
