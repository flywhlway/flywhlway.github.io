PageIndex 开源项目深度调研报告：无向量推理驱动的 RAG 新范式

摘要

随着大语言模型（LLM）在企业级应用中的普及，检索增强生成（RAG）已成为处理私有知识库的核心技术。然而，传统的基于向量相似度的 RAG 在处理长文档、复杂逻辑和专业领域（如金融、法律）时，往往面临精度不足、上下文丢失和黑盒检索等挑战。PageIndex 作为一个新兴的开源项目，提出了一种无向量（Vectorless）、**基于推理（Reasoning-based）**的 RAG 框架。它通过构建文档的层次化树状索引，模拟人类专家的阅读与检索逻辑，实现了极高的检索准确率（在 FinanceBench 上达到 98.7%）。本报告将深入探讨 PageIndex 的技术原理、核心架构、与 LlamaIndex 的综合对比，并提供详尽的实战指南与应用案例。

LlamaIndex 深度调研与应用实践报告

1. 技术原理与核心架构

LlamaIndex（原名 GPT Index）是一个领先的开源数据框架，旨在将私有数据（如 PDF、数据库、API 等）与大语言模型（LLMs）无缝连接。其核心理念是 检索增强生成（RAG）。

1.1 核心架构组件

LlamaIndex 的架构可以抽象为以下四个关键阶段：

阶段 核心组件 功能描述
加载 (Loading) Readers / LlamaHub 从 100+ 数据源（Slack, Notion, PDF）加载原始数据并转换为 Document 对象。
索引 (Indexing) Indices / Nodes 将文档切分为 Node（块），生成 Embedding 并构建可搜索的索引结构（如 VectorStoreIndex）。
存储 (Storing) StorageContext 将索引和元数据持久化到本地磁盘或向量数据库（Pinecone, Milvus, Qdrant）。
