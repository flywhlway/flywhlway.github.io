2. 方案详述（来源：个人网盘部署方案）

下面给你一套家庭 NAS / Ubuntu 22.04 VM 上可长期跑的“个人生产级”方案：pgsty/minio（对象存储 + Console 管理 Bucket） + Filestash（Web 文件管理/个人网盘/图床入口）全 Docker 化部署，并通过你已有的 NPM 反代 + 公网 HTTPS 二级泛域名 + cpolar 对外安全访问。

关键点：

  • MinIO：S3 API(9000) + Console(9001) 分离；反代时建议分别用域名。
  • Filestash：通过管理员密码/htpasswd 做鉴权；并强制绑定外部域名避免 Host 伪造。Filestash 支持用 APPLICATION_URL 指定对外域名（注意不要带 https://，否则可能出现 https://https:// 重定向问题）。 citeturn2search1turn2search3turn2search14
  • MinIO：需要设置 MINIO_BROWSER_REDIRECT_URL 以适配反代后的 Console 访问。 citeturn1search8

0. 你现在这三张图分别说明了什么

  1. Filestash Admin console → Storage Backend
    你已经进到 Filestash 管理台，选择了 S3 存储后端（正确方向）。

  2. Filestash 连接 S3 的弹窗报 “Invalid account”
    这通常不是“账号真的无效”，而是 Filestash 无法用你填的参数成功连到 MinIO 的 S3 API。最常见原因：

  • 你把 MinIO Console 端口 9001 当成 S3 API 端口用了（应是 9000）
  • Endpoint 写法不对 / 走了 https 但 MinIO 没配 TLS
  • Filestash 容器访问不到 MinIO（网络/反代/DNS）
  • MinIO 的 AccessKey/SecretKey 填错（或复制漏了字符）
  • Region / Path-Style / 签名版本不兼容（少见，但也会发生）
  1. MinIO Console → Access Keys
    你已经创建了一个 Access Key（看起来像 service account），这类 Key 非常适合给 Filestash 用 👍
